# database_homework1
Тема: Система аудиту кібербезпеки мережі
Компанія вирішила створити систему аудиту щоб у разі інциденту спеціалістам було легше відстежити на якому моменті. Ми відстежуємо які користувачі підключилися до системи, які пристрої вони використовували та чи були спраби зламати систему.
Логічна структура таблиць:
1. Користувачі вони мають id, name, position, рівень доступу
2. Пристрої вони також мають id, імя пристрою, IP-адреси та статус(безпечний або підозрілий)
3. Події безпеки: ID, Тип події (успішний вхід, помилка пароля, сканування портів), Дата.
4. Логи: ID, ID_Користувача, ID_Пристрою, ID_Події.
5. Результати аудиту: ID, ID_Логу, Оцінка ризику (від 1 до 10), Рекомендація.
